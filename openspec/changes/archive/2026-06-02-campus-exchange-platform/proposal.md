## Why

高校学生群体有大量闲置物品交易需求（教材、数码、宿舍用品等），但现有二手平台（闲鱼、转转）存在物流成本高、信任成本高、缺乏校园归属感等问题。需要一个专为校园场景设计的闲置物品交换平台，降低校内交易的摩擦成本，提供安全、便捷、高效的二手交易体验。本项目为毕业设计/项目实训场景开发，要求达到企业级项目标准，具备完整业务逻辑和可直接运行的代码。

## What Changes

- **新建**: 完整的前后端分离Web应用 — 校园闲置物品交换平台 (Campus Exchange)
- **新建**: 用户端前端 (Vue 3 + TypeScript + Naive UI)，包含首页、搜索、商品详情、发布商品、消息中心、我的页面等6个核心页面
- **新建**: 后台管理前端 (Dashboard布局)，包含数据统计、用户管理、商品管理、举报管理、公告管理等5个管理模块
- **新建**: 后端服务 (Spring Boot 3 + Java 21)，提供完整的RESTful API和WebSocket通信
- **新建**: 数据库设计 (MySQL 8)，不少于16张业务表，完整的外键、索引、逻辑删除、审计字段
- **新建**: 用户模块：注册、登录、JWT认证、资料修改、头像上传、实名认证
- **新建**: 商品模块：发布、编辑、删除、审核、上下架、浏览记录
- **新建**: 搜索模块：关键字搜索、分类筛选、价格筛选、多维度排序
- **新建**: 社交模块：收藏商品、关注用户
- **新建**: 聊天模块：WebSocket私聊、未读消息、已读状态
- **新建**: 订单模块：创建、确认、取消、完成订单，订单状态流转
- **新建**: 评价模块：买卖双方互评、星级评分
- **新建**: 举报模块：举报商品和用户
- **新建**: 通知模块：系统通知、审核通知、订单通知
- **新建**: 文件存储 (MinIO) 集成：商品图片、用户头像、实名认证图片
- **新建**: Redis缓存：热点商品、商品详情、用户信息

## Capabilities

### New Capabilities

- `user-auth`: 用户注册、登录、JWT认证、角色权限管理（普通用户/管理员）
- `user-profile`: 用户资料管理 — 修改资料、修改密码、上传头像、实名认证
- `goods-management`: 商品管理 — 发布、编辑、删除、上下架、审核（管理员）
- `goods-browse`: 商品浏览 — 首页推荐（瀑布流）、最新发布（无限滚动）、商品详情
- `goods-search`: 商品搜索 — 关键字模糊搜索、分类筛选、价格筛选、成色筛选、多维度排序
- `social-interaction`: 社交互动 — 收藏商品、取消收藏、关注用户、取消关注
- `instant-messaging`: 即时聊天 — WebSocket私聊、文本消息、图片消息、未读/已读状态
- `order-trading`: 订单交易 — 创建订单、卖家确认、买家取消、完成订单、订单状态流转
- `user-review`: 用户评价 — 买卖双方互评、1-5星评分、文字评价
- `report-system`: 举报系统 — 举报商品、举报用户、举报原因分类、管理员审核处理
- `notification-system`: 消息通知 — 系统通知、审核通知、订单通知、未读/已读
- `admin-dashboard`: 后台管理 — Dashboard数据统计、用户管理（封禁/解封）、商品管理（审核/下架）、举报管理、公告管理
- `file-storage`: 文件存储 — MinIO集成，商品图片、用户头像、实名认证图片上传与管理

## Impact

- **前端**: 新建完整的 Vue 3 + TypeScript + Vite 项目，使用 Naive UI 组件库、UnoCSS 原子化CSS、Pinia 状态管理、Vue Router 路由
- **后端**: 新建完整的 Spring Boot 3 + Java 21 项目，使用 Spring Security 6 + JWT 认证、MyBatis Plus ORM、Redis 缓存、MinIO 文件存储、WebSocket 通信
- **数据库**: 新建 MySQL 8 数据库，不少于16张表，完整索引和约束设计
- **基础设施**: 需要 MySQL 8、Redis、MinIO 服务可用（开发环境本地运行）
- **无外部API依赖**: 本阶段不集成AI推荐、AI识图、风控系统等第三方服务
